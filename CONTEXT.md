# Monsoon Preparedness & Citizen Assistance — Domain Model

## Core Concepts

**Citizen** — An individual user of the system. Identified by phone number (OTP-verified). May belong to a Family.

**Family** — A group of Citizens sharing a household. Has a primary location (address + GPS), housing type, and composition (adults, children, elderly, persons with disabilities, pets/livestock).

**Location** — Geographic context for a Citizen/Family. Comprises: administrative (state, district, ward/village), GPS coordinates, and hazard zone classification (flood-prone, landslide-prone, cyclone-prone).

**Hazard Zone** — Official classification of an area's vulnerability to specific monsoon hazards. Sourced from IMD, CWC, and municipal data.

**Preparedness Phase** — Temporal stage of monsoon cycle:

- **Pre-Monsoon** (Apr–May): Preparation, home hardening, kit assembly
- **Active Monsoon** (Jun–Sep): Real-time alerts, evacuation, shelters, SOS
- **Post-Monsoon** (Oct–Nov): Recovery, damage assessment, disease prevention

**Preparedness Plan** — Personalized, phase-aware action plan for a Family. Generated by GenAI from: Family profile, Location hazard profile, current Phase, and real-time weather.

**Checklist Item** — Atomic action in a Preparedness Plan. Has: description, priority (critical/high/medium/low), phase applicability, completion status, and optional location-specific guidance.

**Weather Alert** — Official warning from IMD/CWC. Contains: alert type (heavy rain, cyclone, flood, landslide), severity (watch/alert/warning), affected districts, validity period, and recommended actions.

**Localized Alert** — Weather Alert translated and contextualized by GenAI for a specific Citizen/Family in their preferred language, with location-specific actionable guidance.

**Shelter** — Designated emergency shelter. Has: name, address, GPS, capacity, facilities (water, toilets, medical), current occupancy, and managing authority.

**Evacuation Route** — Path from a Location to a Shelter. Has: waypoints, road conditions, estimated travel time, and suitability for vehicle/pedestrian.

**SOS Request** — Emergency signal from a Citizen. Contains: Citizen ID, GPS location, timestamp, emergency type (medical, trapped, evacuation needed), and status (active/responded/resolved).

**Community Report** — Crowdsourced observation from a Citizen. Types: waterlogging, road closure, landslide, shelter status, power outage. Has: location, severity, timestamp, reporter credibility, verification status.

**Vulnerability Index** — Composite score for a Location derived from: historical flood data, drainage capacity, elevation, population density, infrastructure quality, and socio-economic indicators (SECC/census).

**Risk Score** — Dynamic, personalized score for a Family at a given moment. Computed from: Weather Alerts, Vulnerability Index, Family composition (elderly, children, disabilities), housing type, and preparedness completion.

**GenAI Agent** — LLM-powered conversational interface. Capabilities: RAG over official advisories (IMD, NDMA, ULBs), personalized plan generation, multilingual Q&A, risk explanation, image analysis (waterlogging severity), voice interaction.

**Channel** — Delivery mechanism for citizen-facing features. Types: PWA (primary), WhatsApp Business API, SMS/USSD (critical alerts only), IVR/voice (low-literacy).

**Offline Cache** — Pre-fetched data stored on device for offline use. Includes: personalized checklists, evacuation routes, shelter locations, emergency contacts, basic first-aid guidance.

**Data Source** — External system providing authoritative data. Types: IMD (weather), CWC (rivers/reservoirs), ULB/Municipal (flood maps, shelters, drainage), Discoms (power outages), Transport (rail/road/flight disruptions).

## Relationships

- Citizen ↔ Family: Many-to-one (a Citizen belongs to one Family; a Family has many Citizens)
- Family ↔ Location: One-to-one (a Family has one primary Location)
- Location ↔ Hazard Zone: Many-to-one (a Location falls in one or more Hazard Zones)
- Family ↔ Preparedness Plan: One-to-one per Phase (a Family has one active Plan per Phase)
- Preparedness Plan ↔ Checklist Item: One-to-many
- Weather Alert → Localized Alert: One-to-many per Citizen/Family (each Alert generates personalized Localized Alerts)
- Location ↔ Shelter: Many-to-many (a Location maps to multiple Shelters; a Shelter serves multiple Locations)
- Location ↔ Evacuation Route: One-to-many (a Location has multiple Routes to different Shelters)
- Citizen → SOS Request: One-to-many
- Citizen → Community Report: One-to-many
- Location ↔ Vulnerability Index: One-to-one
- Family + Weather Alert + Vulnerability Index → Risk Score: Computed per Family per Alert event
- GenAI Agent ↔ Data Source: Reads from (RAG) and writes to (alerts, plans)
- Channel ↔ Citizen: One-to-many (a Citizen may use multiple Channels)
- Citizen ↔ Offline Cache: One-to-one (per device)

## Key Decisions (for ADR consideration)

1. **Phone OTP only auth** — Minimal PII, passwordless, accessible. Trade-off: no social login, account recovery via re-OTP only.

2. **PWA as primary channel** — Installable, offline-capable, single codebase. Trade-off: no native push notifications on iOS (workaround: SMS/USSD for critical alerts).

3. **API-based GenAI (OpenAI/Anthropic)** — Fastest for hackathon. Trade-off: cost, latency, data privacy (mitigated by minimal PII).

4. **Modular monolith architecture** — Single deployable unit, internal service boundaries. Trade-off: scaling limitations beyond pilot.

5. **12 Indian languages + dialects** — Inclusive but increases GenAI prompt complexity and translation QA burden.

6. **Offline-first checklists & maps** — Critical for monsoon power/internet outages. Trade-off: storage size, cache freshness.

7. **Hackathon scope: 1 district, 2 hours** — Extreme time constraint drives ruthless prioritization: PWA only, no mock data for IMD/CWC/ULB, GenAI via API, no real SOS integration.
